# Bright-coffee-shop-dashboard

Dashboard

AIM OF THE PROJECT

Is to build a single, visual sales dashboard that turns raw point-of-sale records into actionable answers about:
• Revenue trends and seasonality
• Top-selling products and categories
• Store-level performance
• Customer demographics & buying hours


PROBLEM STATEMENT

A coffee shop chain operating across three locations had raw transaction data but no consolidated view of which products, stores, time-periods or customer segments were driving revenue.
Decision-makers struggled to plan inventory, staffing and promotions without trusted, easy-to-read insights.


OBJECTIVES - STEPS TAKEN TO SOLVE THE AIM
COLLECTION
- Import raw POS transaction data (csv/pdf).

CLEAN
- Removed duplicates, fixed types, standardise dates & currency to rands

EXPLORE
- Profiled revenue,transactions,products,stores,hours

MODEL
- Aggregated KPIs: monthly revenue, category mix, top-N products.

VISUALISE
- Built charts: line, bar, doughnut, area for each question.

REPORT
- Packaged findings into this executive PowerPoint deck
